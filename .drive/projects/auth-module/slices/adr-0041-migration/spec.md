# Slice spec: params-vocabulary settlement + auth migration onto ADR-0042

> Status: vocabulary settled with Will 2026-07-24 (conversation record); this
> spec is exhaustive by intent. Every name, type, file placement, and behavior
> is pinned. An implementer who finds a genuine gap reports it — they do not
> improvise. Doc authoring (§ 7) is orchestrator work, not implementer work.

## 1. The settled vocabulary (binding; the anchor for every edit below)

**There is one concept: a param, bound at wiring time to a source.** A source
is target-provided logic that turns the parameter information the user
supplies at wiring time into a value. The sources:

| Source | User supplies at wiring | Value comes from | In Alchemy state? | The one "secret"? |
|---|---|---|---|---|
| literal | the value itself | the binding | yes (it's in the input document) | no |
| `envParam(NAME)` | the env var name | the environment, per stage | **never** | no |
| `envSecret(NAME)` | the env var name | the platform secret store, per stage | **never** | **yes — the only one** |
| `generatedParam(...)` | generation parameters | **generated by the target at deploy** | **yes** (that is the point: stable across redeploys) | no |

Consequences, not categories:

- **Persistence follows the source.** Env/secret sources re-resolve per
  environment — deliberately, so values vary by environment, not by
  deployment logic; their values never enter Alchemy state. A generated
  source produces its value once at deploy and persists it in state.
- **Redaction is an orthogonal facet** (printed vs not), independent of
  source. `envSecret` is always redacted. A generated param may be redacted
  (the auth signing value) or not.
- **"Secret" means exactly one thing:** `envSecret` — externally-sourced,
  user-provided, wired by name, value never in state. The framework never
  produces a secret. "Minted secret" and "config secret" do not exist.
- **The word "minted" is retired** repo-wide in this domain. The value is
  *generated*. (ADR-0030's prose may keep "mint" as a verb until its
  rewrite lands in this slice; after this slice, no API name, type, resource
  kind, or new prose uses "minted".)

`PrismaCloud.MintedSecret` regenerates the value; acceptable pre-merge, noted
in the PR body.

## 3. `generatedParam()` — the API (target package, `@internal/prisma-cloud`)

File: `packages/1-prisma-cloud/1-extensions/target/src/param.ts` (beside
`envParam` — it is a param source, NOT in `secret.ts`).

```ts
/** Wiring info for a target-generated param value. */
export interface GeneratedParamOptions {
  /** Byte length of the generated value (base64-encoded). Default 32. */
  readonly bytes?: number;
  /** Redacted: boot wraps the value in the redacting box; the deploy report never prints it. Default true. */
  readonly redacted?: boolean;
}
export function generatedParam(opts?: GeneratedParamOptions): ParamSource<GeneratedParamPayload>;
```

- Built on core's existing `paramSource()` SPI (ADR-0032's frame: env
  sourcing is a target source; generated is a second target source). Payload
  carries a unique-symbol brand (same pattern as `EnvSecretPayload` /
  the env param payload) plus `{ bytes: number; redacted: boolean }` with
  defaults applied at construction.
- `isGeneratedParamSource(value): value is ParamSource<GeneratedParamPayload>`
  exported for the serializer, preflight, and tests.
- Exported from `target/src/exports/index.ts` and re-exported on the public
  package exactly where `envParam` is.
- Validation: `bytes` must be an integer ≥ 16 and ≤ 1024 — construction
  throws otherwise (message names the bound). No other options in v1.

## 4. Deploy mechanics (ADR-0042 world — the branch this lands on)

All on the post-#161 shape (`serializer.ts` input walk, `preflight.ts`
`collectSecretLeafNames`, `descriptors/compute.ts` serialize step).

1. **Binding walk (`serializer.ts` `resolveInputBinding`).** Recognize
   `isGeneratedParamSource(value)` as a leaf BEFORE the param/secret source
   branches. A generated leaf resolves to a **pointer**, exactly parallel to
   the `$secret` pointer, with its own key: the document carries
   `{"$generated": "<VAR>", "redacted": <bool>}` where `<VAR> =
   generatedParamVarName(address, path)` = `COMPOSER_<addr>_<KEY>_GENERATED`
   (`<KEY>` = the input key path, serializer-normalized the same way secret
   pointer names are). **The pointer carries the `redacted` facet** — the
   deploy knows it (from the leaf), boot must not, because boot is
   schema-blind (a Standard Schema exposes only `validate`, never its field
   types). Boot reads the facet off the pointer, not the schema (amended
   2026-07-24, D1: the earlier "box when the schema field expects one" was
   not implementable). For schema validation at deploy, the sentinel is: a
   `SecretString` box (`new SecretBox('')`) when `redacted: true`; the empty
   string `''` when `redacted: false`. The walk also appends to a new
   `ResolvedInputBinding.generated: readonly GeneratedLeaf[]` list:
   `{ varName: string; bytes: number; redacted: boolean; path: string }`.
   The walk itself creates NO resource — it stays pure; ADR-0042's
   "the walk mints no platform resource" sentence is amended (§ 7.5) to
   "the walk creates no platform resource; generated leaves are recorded
   for the descriptor's deploy step, which provisions them."
2. **Escaping.** The literal object `{"$generated": ...}` appearing as a
   user literal is escaped the same way `$secret` collisions are (mirror the
   existing `$secret` escaping mechanism byte-for-byte — same function,
   extended table).
3. **Descriptor deploy step (`descriptors/compute.ts`).** After
   `serializeInput` returns, iterate `resolved.generated`: for each leaf,
   `yield* GeneratedParam(resourceId, { bytes })` (the Alchemy resource,
   § 5) then `yield* Prisma.EnvironmentVariable(varName, resource.value)`.
   Resource id = `` `${inputDocumentRowKey}:${path}-generated` `` — stable
   per service+key so reconcile keeps the value.
4. **Preflight (`preflight.ts`).** `collectSecretLeafNames`'s recursion
   skips generated leaves (`if (isGeneratedParamSource(v)) continue;`)
   — nothing for an operator to seed, nothing to check. Generated leaves
   never appear in the missing-bindings error.
5. **Boot (`readInput`/`hydrateInputDocument`).** A
   `{"$generated": VAR, "redacted": <bool>}` pointer hydrates from
   `process.env[VAR]`, wrapping in the redacting box iff the pointer's
   `redacted` is `true`, plain string otherwise. Boot reads the facet from
   the POINTER, never the schema (§ 4.1). A `redacted: true` generated leaf
   whose schema field is a plain string, or a `redacted: false` leaf whose
   field is `secretString()`, is a wiring error the schema catches at
   validation — the pointer and the schema agree by construction because the
   same author writes both. A missing VAR at boot is the same hard error a
   missing `$secret` var is, with a message naming the var and "the deploy
   provisions this variable — its absence means the deploy and the running
   service disagree".
6. **Deploy report.** The input document prints as today; a generated leaf
   prints as its pointer (`{"$generated":"COMPOSER_…"}`), never a value —
   true for redacted AND unredacted generated params (the value is in
   state, not in the document). The report's absent-keys listing is
   unaffected.
7. **Collision guard (closes review finding F02 properly).** After the walk,
   assert all pointer var names across the service — `$secret` names AND
   `$generated` names — are distinct after the serializer's key
   normalization (uppercasing included). Duplicate → deploy error naming
   both input paths. One general check; no `_GENERATED`-specific special
   case. (This also covers the pre-existing `foo`/`FOO` class for secrets.)

## 5. The Alchemy resource

File `packages/1-prisma-cloud/1-extensions/target/src/generated-param-resource.ts`
(rename of `minted-secret-resource.ts`; mechanics unchanged):

- Resource type string `'PrismaCloud.GeneratedParam'`; props `{ bytes: number }`.
- Mint→generate: `crypto.getRandomValues` (Web Crypto, no `node:` import),
  `bytes` random bytes, base64. `reconcile: ({ output }) => output ?? generate()`
  (stable across redeploys; changing `bytes` on an existing resource keeps
  the old value — rotation is destroy/recreate; document in the doc comment).
- `list → []`, `delete → no-op`. Provider registered in
  `control/extension.ts` `providers()` as `GeneratedParamProvider()`.

## 6. Auth migration (applies the compatibility map; every item pinned)

`bot/claude/adr-input-schema` (#161's tip), then S1 content re-applied, then


   The field name stays `secret` — it is Better Auth's own option name for
   its instance secret; local vocabulary, not framework vocabulary. The
   schema field uses `secretString()` because that IS the redaction facet in
   an ADR-0042 schema: the field type says "boot hands me a redacting box".


   `@internal/prisma-cloud`.

   ```ts
   const { db, email } = service.load();
   const { baseUrl, secret } = service.input();
   const port = service.port();
   ```

5. **`examples/auth/module.ts`**: no change (module-boundary bindings).
6. **`examples/auth/src/{api,ops}/server.ts`**: `const { port } =

## 7. Documentation edits (orchestrator-authored; same PR; pinned content)

1. **`docs/design/03-domain-model/glossary.md`** — the Configuration
   section is rewritten to the § 1 model: param + source; the four sources;
   persistence follows the source; redaction is a facet; `envSecret` is the
   only secret; the framework never produces a secret, only config params.
2. **`docs/design/10-domains/config-params.md`** — the "three out-of-band
   value channels" paragraph is rewritten to the source table; "framework-
   minted" → "target-generated"; a new short § Generated sources subsection
   (deploy-time generation, persistence, stability, redaction facet,
   rotation = destroy/recreate).
3. **ADR-0029** — scope sentence added: "This ADR governs *secrets*:
   externally-sourced, user-provided values (`envSecret`). Values the
   target generates are params (ADR-0032's source frame; ADR-0042's
   `$generated` pointer) — they are not secrets and MAY be recorded in
   deploy state." The "never carries a value" invariant is restated as
   scoped to secrets.
4. **ADR-0030** — terminology pass: the service key is *a generated param
   before that concept had a name*; "mint" → "generate" in normative text;
   the rejected alternative "a real ADR-0029 secret" gains: "(a generation
   path now exists — `generatedParam()`, ADR-0042 § Generated sources — and
   the decision stands: a service key is config the system maintains, not
   an operator-provided secret)".
5. **ADR-0042** — new § "Generated sources": the fourth leaf, the
   `$generated` pointer, the descriptor-provisioned resource, the amended
   walk sentence (§ 4.1), the redaction-via-schema-type rule, the collision
   guard. Plus the leaves list ("literals, `envParam(...)`, or
   `envSecret(...)`") gains `generatedParam(...)`.
6. **Consistency checks (no rewrite expected):** ADR-0031 (its
   "framework-minted provision needs" vocabulary gets a pointer to the new
   terms), ADR-0032 (generated cited as a second target source).
